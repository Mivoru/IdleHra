Architectural Critique of Sprint 71
1. The Async Connection Multiplexer Deadlock Gate
The OfflineStateEngine.cs executes calculations inside a strict System.Data.IsolationLevel.Serializable database transaction block. If an account with multiple character slots logs back in after an extended absence, the engine fires concurrent state evaluation sub-routines. Because the custom socket stack handles login parsing on the shared HttpListener thread pool, a wave of players authenticating concurrently will cause cross-row database locks to cascade. This results in connection pool starvation, thread pool exhaustion, and intermittent transaction deadlocks that drop player sessions before the state payload can flush.

2. Time Refunding Precision Loss
When inventory capacity saturates mid-calculation, overflow time is converted into BankedChronoSeconds. If this time slice fraction contains sub-second remainders, truncating the duration to an integer value before the final check-constraint update bleeds small fractions of calculated idle progression. This breaks total resource determinism across prolonged offline cycles.
