Architektonické vady a rizika implementace
1. Konflikt kódování: UTF-8 vs. TextMeshPro (Kritická chyba)
Problém: Plán navrhuje ukládat lokalizační data jako surové UTF-8 bajty (byte*) a předávat je přímo do TMP_Text.SetCharArray. TextMeshPro v Unity však interně pracuje výhradně s UTF-16 (char – 2 bajty) nebo s Unicode code points (int – 4 bajty). Pokud do SetCharArray napumpuješ surový UTF-8 stream, vykreslovací jádro TMP interpretuje jednotlivé bajty jako samostatné znaky. Výsledkem bude kompletně rozsypaný text (mojibake) a okamžité selhání zobrazení u českých nebo německých diakritických znaků (např. č, ü).

Náprava: LocalizationMatrix musí držet lokalizační archy v unmanaged paměti přímo jako UTF-16 znaky (char*). Lookup bude vracet pointer na char, který lze přes unmanaged blokovou kopii (Unsafe.CopyBlock) bez alokací přelít do předalokovaného klientského bufferu typu char[], který již TMP_Text.SetCharArray nativně akceptuje.

2. Úniky paměti v Unity Editoru (Domain Reload Leaks)
Problém: FlightRecorder alokuje 64 KB nativní haldy přes Marshal.AllocHGlobal. Statické ukazatele v Unity při restartu Play Mode (Domain Reload) sice ztratí své reference v C# kontextu, ale paměť alokovaná v operačním systému zůstane viset. Po několika desítkách spuštění hry v editoru dojde k tichému vyčerpání paměti (Native Memory Leak).

Náprava: Třída musí explicitně implementovat uvolňovací sekvenci registrovanou přes Unity atribut:

C#
[RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.SubsystemRegistration)]
private static void OnEditorCleanup() { /* Marshal.FreeHGlobal if allocated */ }
3. Chybějící vláknová bezpečnost (Thread-Safety Buffer Race)
Problém: Síťový klient (NetworkClient) běží na pozadí v samostatném threadu (příjem WebSocket dat). Pokud síťové vlákno zapisuje hlavičky paketů do FlightRecorder ve stejný okamžik, kdy hlavní vykreslovací vlákno (Main Thread) čte buffer pro zobrazení telemetrie na plátno, dojde k poškození indexů a datovému souběhu (Race Condition).

Náprava: Zápisový index kruhového bufferu musí být striktně atomický. Pozice se musí posouvat výhradně přes Interlocked.Increment.

Architektonická kritika a skrytá rizika Sprintu 57
1. Riziko narušení paměti při Unsafe.CopyBlock (Heap Corruption Vector)
Problém: Pokud metoda Lookup vrátí počet znaků (charCount), který překračuje délku pre-alokovaného spravovaného pole char[] v UI binderu, Unsafe.CopyBlock provede zápis mimo alokované hranice pole. V C# to nepovede k bezpečné IndexOutOfRangeException, ale k okamžité korupci spravované haldy (Managed Heap Corruption) nebo k pádu aplikace na AccessViolationException.

Opatření: Klientská metoda Lookup musí bezpodmínečně přijímat parametr maxTargetLength a interní kopírování omezit přes Math.Min(charCount, maxTargetLength).

2. Teardown fáze a signed modulo wrapping ve FlightRecorder
Problém: Pokud index zapisovače roste přes Interlocked.Increment do nekonečna, po dosažení int.MaxValue přeteče do záporných čísel. Pokud implementace provádí rotaci indexu operací modulo se znaménkem (index % bufferSize), záporný index vyvolá při přístupu do pole havárii.

Opatření: Velikost kruhového bufferu musí být striktně mocninou dvojky (např. 1024 prvků) a rotace indexu musí používat bitový operátor AND (index & (bufferSize - 1)), který je imunní vůči znaménkovému přetečení a je řádově rychlejší než operace modulo.

// Engine/BillingVerificationEngine.cs
using System;
using System.Threading.Tasks;
using Microsoft.EntityFrameworkCore;
using FolkIdle.Server.Models;

namespace FolkIdle.Server.Engine
{
    public class BillingVerificationEngine
    {
        private readonly IDbContextFactory<FolkIdleDbContext> _contextFactory;
        private readonly RedisSessionCache _redisCache;

        public BillingVerificationEngine(IDbContextFactory<FolkIdleDbContext> contextFactory, RedisSessionCache redisCache)
        {
            _contextFactory = contextFactory;
            _redisCache = redisCache;
        }

        public async Task<bool> VerifyPurchaseAsync(long playerId, string transactionId, string productId, int premiumAmount)
        {
            using var context = await _contextFactory.CreateDbContextAsync();
            using var transaction = await context.Database.BeginTransactionAsync(System.Data.IsolationLevel.Serializable);

            try
            {
                var existing = await context.PrimaryPurchaseLedgers
                    .SystemLockForUpdate()
                    .FirstOrDefaultAsync(p => p.TransactionId == transactionId);

                if (existing != null) return false;

                var profile = await context.PlayerRecords
                    .SystemLockForUpdate()
                    .FirstOrDefaultAsync(p => p.PlayerId == playerId);

                if (profile == null) return false;

                int previousBalance = profile.PremiumCurrency;
                profile.PremiumCurrency += premiumAmount;

                var purchase = new PrimaryPurchaseLedger
                    {
                        TransactionId = transactionId,
                        PlayerId = playerId,
                        ProductId = productId,
                        PurchaseState = 1,
                        TimestampProcessed = DateTimeOffset.UtcNow.ToUnixTimeSeconds()
                    };

                var log = new EventHorizonPremiumLedger
                    {
                        TransactionId = transactionId,
                        PlayerId = playerId,
                        PreviousBalance = previousBalance,
                        NewBalance = profile.PremiumCurrency,
                        Timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds()
                    };

                context.PrimaryPurchaseLedgers.Add(purchase);
                context.EventHorizonPremiumLedgers.Add(log);

                await context.SaveChangesAsync();
                await transaction.CommitAsync();

                return true;
            }
            catch
            {
                await transaction.RollbackAsync();
                throw;
            }
        }

        public async Task HandleRefundAlertAsync(string transactionId)
        {
            using var context = await _contextFactory.CreateDbContextAsync();
            using var transaction = await context.Database.BeginTransactionAsync(System.Data.IsolationLevel.Serializable);

            try
            {
                var purchase = await context.PrimaryPurchaseLedgers
                    .SystemLockForUpdate()
                    .FirstOrDefaultAsync(p => p.TransactionId == transactionId);

                if (purchase == null || purchase.PurchaseState == 2) return;

                purchase.PurchaseState = 2;

                var profile = await context.PlayerRecords
                    .SystemLockForUpdate()
                    .FirstOrDefaultAsync(p => p.PlayerId == purchase.PlayerId);

                if (profile != null)
                {
                    int previousBalance = profile.PremiumCurrency;
                    int deduction = 1000; // Mapped via product lookup matrix
                    profile.PremiumCurrency -= deduction;

                    var log = new EventHorizonPremiumLedger
                        {
                            TransactionId = transactionId,
                            PlayerId = purchase.PlayerId,
                            PreviousBalance = previousBalance,
                            NewBalance = profile.PremiumCurrency,
                            Timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds()
                        };
                    context.EventHorizonPremiumLedgers.Add(log);

                    if (profile.PremiumCurrency < 0)
                    {
                        profile.IsQuarantined = true;
                        await _redisCache.SetQuarantineFlagAsync(purchase.PlayerId, true);
                        
                        await context.Database.ExecuteSqlRawAsync(
                            "UPDATE \"MarketOrderRecords\" SET \"IsQuarantined\" = TRUE WHERE \"PlayerId\" = {0} AND \"IsActive\" = TRUE", 
                            purchase.PlayerId);
                    }
                }

                await context.SaveChangesAsync();
                await transaction.CommitAsync();
            }
            catch
            {
                await transaction.RollbackAsync();
                throw;
            }
        }
    }
